今日はjQuery Deferredについて書きます。

jQuery Deferredってそもそも何かということについては、以下のサイトが一番わかりやすくまとめられてます。

結局jQuery.Deferredの何が嬉しいのか分からない、という人向けの小話

ここではFacebookページの投稿に対するいいね・コメント・シェア数をAjaxで取得して全部取得したら表示する…的なことを書いています。

Deferredの$.whenを利用すると複数の非同期通信が全て終わったら何か処理するということが書けるんですが、公式ドキュメントの例だと、

$.when($.ajax({ 処理1 }), $.ajax({ 処理1 }))

のように処理の回数が一定のものしか載っていなかったので、$.eachなどでループさせるなどリクエスト数が可変の際に実現する方法を調べました。 

$.when.apply($, promises)
  .done(function() {
       // do successed
  })
  .fail(function() {
       // do failed
  });

それがここの部分。

applyは第一引数にapplyを実行する関数を呼び出すthis参照値を渡し、

第二引数に配列を渡すと、渡した配列を引数にapplyを実行した関数を呼び出してくれる…らしい(笑) 

$.when.apply($, [deferred, deferred])
  .done(function() {
       // do successed
  })
  .fail(function() {
       // do failed
  });

つまり、第二引数にループして返ってきたdeferredオブジェクトの配列を突っ込めば、

任意の数のリクエストでも対応できます！便利！
