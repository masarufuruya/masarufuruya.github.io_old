Non-Fast-Forwardって結局何？という人向けのまとめ

2014年01月18日

Tweet

最近、マージでミスをして他人のコードを消してしまい自分のGit力の低さに

身の危険を感じたので改めてGitを勉強し直しました。

今回はまず、Fast-ForwardとNon-Fast-Forwardの違いについてまとめたいと思います。

Fast Forwardとは？

ある時点でmasterブランチからブランチAが出来たとします。その後、ブランチAでコミットをしていきます。

// 画像

ブランチAで作業している間、masterブランチでは全く変更が行われていないとします。

この状態ならブランチAをmasterブランチに安全にマージすることが出来ます。

ブランチAでコミットした内容をmasterブランチにもコミットすればいいだけなので。

ただ、実際にはFast ForwardマージとはブランチAでコミットした内容を再度コミットするようなことはしません。

単にmasterブランチの先頭をブランチAの先頭へポインタするように変更するだけです。

これが、Fast Forward マージです。

Non Fast Forwardとは？

これに対して、Non Fast Forwardとは、、

// 画像

ブランチAが変更されている間、masterブランチも変更されている場合

この場合は、ただ単にポインタを移動するだけでは変更内容を取り込むことは出来ません。

masterブランチとブランチAで変更された内容をそれぞれ活かしたまま、

それらをマージして新たな合流地点をつくる、それがNon Fast Forwardマージになります。

ローカルブランチの変更箇所をプッシュする前にリモートブランチが変更された場合

この状態でローカルのコミットをプッシュしようとすると、

Fast Forward Push

Non Fast Forward Push
