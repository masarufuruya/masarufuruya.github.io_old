たった10分！Middelman+GithubPagesを使ってブログを始めてみよう

2014年01月20日

Tweet

最近、ブログを作る方法を聞かれることが何度かあったので、

本ブログの構成であるMiddleman + Github Pagesを使ったブログの構築手順をまとめたいと思います。

Middleman,Github Pagesに関する詳しい説明は省きます。

GithubにGithub Pages用のリポジトリを作成する

まず、Githubへログインして、自分のユーザーネーム.github.ioというリポジトリを作成します。

作成が完了したら、リポジトリの設定画面にGithub Pagesという項目があるので、

ここの[Automatic Page Generator]というボタンをクリックします。

すると、勝手にGithub上にサイトがホスティングされるようになります。

http://username.github.ioにアクセスするとサイトにアクセスできます。

Middlemanをインストール

次に静的ジェネレータであるMiddlemanをMacにインストールします。

$ gem install middleman

プロジェクトフォルダをつくる

まずは適当な場所にMiddlemanのプロジェクトフォルダを作成します。

middleman init blog

作成したプロジェクトフォルダへ移動します。

cd blog

リモートリポジトリを登録する

git init
git remote add origin git@github.com:username/username.github.io.git

ちゃんと登録出来たか確認するには、以下のコマンドを使います。

git remote -v

これでMiddlemanで作成したファイルをGitで管理して、

プッシュすればGithubPagesに公開できるようになります。

ローカルサーバーで動作を確認する

いちいちプッシュせずに、ローカルでサイトの作成状況を確認したいという場合には、

以下コマンドでmiddlemanサーバーを立ち上げて確認します。

bundle exec middleman server

http://0.0.0.0:4567にアクセスすると、middlemanが立ち上がっているのがわかると思います。

後はMiddlemanとBootstrapを活用してサイトを構築していくだけでサクッとブログを立ち上げることが出来ます。

次回は具体的なMiddelmanの使い方などを紹介したいと思います。
